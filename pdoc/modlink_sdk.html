Base class for all ModLink drivers.

A driver instance represents one host-managed device endpoint. The host loads the driver through a zero-argument factory, reads its static metadata, and then executes lifecycle methods on a dedicated worker thread.

General contract:

  • device_id must be stable and non-empty.
  • descriptors() must describe every stream the driver may emit.
  • control methods are synchronous and run on the driver worker thread.
  • start_streaming() should return quickly after acquisition starts.
  • one driver instance manages at most one active device connection.

150class LoopDriver(Driver):
151    """Convenience base class for timer-driven loop-style drivers.
152
153    ``LoopDriver`` is intended for common polling-style devices where one
154    short method can be called repeatedly to check for fresh data. It already
155    satisfies the normal ``Driver`` streaming contract, so it works with the
156    existing runtime without any core changes.
157
158    Typical subclass responsibilities:
159
160    - implement ``search()``, ``connect_device()``, ``disconnect_device()``,
161      and ``descriptors()`` as usual
162    - implement ``loop()`` for one short iteration
163    - optionally override ``loop_interval_ms``
164    - optionally implement ``on_loop_started()`` and ``on_loop_stopped()``
165    """
166
167    loop_interval_ms = 10
168    """Driver-level timer cadence used by ``start_streaming()``.
169
170    This is not a stream sample rate. A single loop may emit zero, one, or
171    multiple payloads, and different streams may still run at different
172    effective rates within the same driver.
173    """
174
175    def __init__(self) -> None:
176        super().__init__()
177        self._loop_timer: QTimer | None = None
178
179    def on_thread_started(self) -> None:
180        """Create the loop timer on the driver worker thread."""
181        self._loop_timer = QTimer(self)
182        self._loop_timer.timeout.connect(self.loop)
183
184    def start_streaming(self) -> None:
185        """Start timer-driven looping.
186
187        The default implementation calls ``on_loop_started()`` once and then
188        schedules repeated ``loop()`` calls on the driver thread.
189        """
190        if self._loop_timer is None:
191            raise RuntimeError("loop timer is not ready")
192        if self._loop_timer.isActive():
193            return
194
195        self.on_loop_started()
196        self._loop_timer.start(int(self.loop_interval_ms))
197
198    def stop_streaming(self) -> None:
199        """Stop timer-driven looping.
200
201        The default implementation stops future ``loop()`` calls and then
202        calls ``on_loop_stopped()`` once.
203        """
204        if self._loop_timer is None or not self._loop_timer.isActive():
205            return
206
207        self._loop_timer.stop()
208        self.on_loop_stopped()
209
210    def on_loop_started(self) -> None:
211        """Optional hook called once before the loop timer starts."""
212
213    def on_loop_stopped(self) -> None:
214        """Optional hook called once after the loop timer stops."""
215
216    def loop(self) -> None:
217        """Run one short loop iteration.
218
219        The method should stay short and non-blocking. It may emit one or more
220        frames, or no frames if no fresh data is available. Think of this as
221        an Arduino-style ``loop()`` running on the driver thread.
222        """
223        raise NotImplementedError(f"{type(self).__name__} must implement loop")
